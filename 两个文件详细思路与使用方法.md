# `pickisainstrclass.py` 与 `pickinstrtype.py` 详细思路与使用方法

本文整理这两个文件的职责、核心流程，以及它们运行时依赖外部程序提供的参数/状态。

---

## 1. `pickisainstrclass.py`

### 1.1 作用
- 负责“先选指令大类（`ISAInstrClass`）”。
- 这是随机生成一条指令前的第一层调度。
- 在静态权重基础上，结合动态反馈（Bug/Coverage/Novelty/被选次数衰减）做概率调整。

### 1.2 核心流程（`gen_next_isainstrclass(fuzzerstate)`）
1. 读取 `fuzzerstate.isapickweights` 作为初始权重。
2. 按设计能力/当前特权态/容错策略过滤不可用类别（如无 FPU、无 64 位能力、CSR 受限等）。
3. 根据寄存器状态过滤敏感类别（如没有 `CONSUMED` 寄存器时禁用若干类别）。
4. 合并动态反馈分数：
   - `isaclass_bug_scores`
   - `isaclass_coverage_scores`
   - `isaclass_novelty_scores`
   - `isaclass_selection_counts`
5. 用 ε-贪心策略选类：
   - 概率 `ISACLASS_EPSILON_EXPLORATION` 走纯随机探索；
   - 否则按加权随机采样。

### 1.3 外部程序必须提供的参数/状态

#### A. 函数输入
- `fuzzerstate`（核心上下文对象，需包含下列字段/子对象）。

#### B. `fuzzerstate` 必备字段（被本文件直接访问）
- `isapickweights`：`dict[ISAInstrClass, float]`，初始类别权重。
- 设计能力相关：
  - `is_design_64bit`
  - `design_has_fpu`
  - `design_has_fpud`
  - `is_fpu_activated`
  - `design_has_muldiv`
  - `design_has_amo`
  - `design_has_supervisor_mode`
  - `design_name`
- 特权相关：
  - `authorize_privileges`
  - `privilegestate`（至少含 `privstate`、`is_mtvec_populated`、`is_stvec_populated`、`is_mepc_populated`、`is_sepc_populated`，并支持 `is_ready_to_take_exception(fuzzerstate)`）
- 寄存器状态相关：
  - `intregpickstate`（至少支持 `get_num_regs_in_state(...)`、`exists_reg_in_state(...)`）
- 统计相关：
  - `special_instrs_count`

#### C. 可选动态反馈字段（缺省会退化为静态策略）
- `isaclass_bug_scores`
- `isaclass_coverage_scores`
- `isaclass_novelty_scores`
- `isaclass_selection_counts`

#### D. 其他外部模块参数（由外部环境配置）
- `params.runparams.DO_ASSERT`
- `params.fuzzparams.NUM_MIN_FREE_INTREGS`
- `params.fuzzparams.MAX_NUM_FENCES_PER_EXECUTION`
- `cascade.toleratebugs` 中若干容错开关函数返回值

## 2. `pickinstrtype.py`

### 2.1 作用
- 在“已选定指令大类 `isaclass`”后，选择该类别下的具体指令字符串（如 `add`、`fdiv.s` 等）。
- 支持设计特定禁用逻辑（如 `vexriscv`、`cva6`）。
- 支持按动态反馈对单条指令概率做增益与衰减。

### 2.2 核心流程（`gen_next_instrstr_from_isaclass(isaclass, fuzzerstate)`）
1. 从 `INSTRTYPE_INITIAL_RELATIVE_WEIGHTS[isaclass]` 拷贝该类下所有指令的初始权重。
2. 若目标是 `vexriscv`，调用 `forbid_vexriscv_ops(...)` 禁用/恢复部分浮点相关指令。
3. 若 `isaclass == SPECIAL`，递增 `fuzzerstate.special_instrs_count`。
4. 若目标是 `cva6`，按容错开关禁用部分浮点除法/开方/转换指令。
5. 应用动态反馈：
   - `instr_bug_scores`
   - `instr_coverage_scores`
   - `instr_novelty_scores`
   - `instr_selection_counts`
6. ε-贪心采样具体指令：
   - 有效候选为空时回退到该类初始候选；
   - 否则按探索/利用策略选取。

### 2.3 外部程序必须提供的参数/状态

#### A. 函数输入
- `isaclass: ISAInstrClass`（且不能是 `REGFSM/FPUFSM/EXCEPTION/TVECFSM/PPFSM/EPCFSM` 这些特殊类）。
- `fuzzerstate`。

#### B. `fuzzerstate` 必备字段
- `design_name`
- `special_instrs_count`（当 `isaclass == SPECIAL` 时会写入+1）

#### C. 可选动态反馈字段（缺省会退化为静态策略）
- `instr_bug_scores`
- `instr_coverage_scores`
- `instr_novelty_scores`
- `instr_selection_counts`

#### D. 其他外部模块参数（由外部环境配置）
- `params.runparams.DO_ASSERT`
- `params.fuzzparams.NUM_MIN_FREE_INTREGS`（本文件导入但当前流程中未直接使用）
- `cascade.toleratebugs` 中与 `vexriscv/cva6` 相关的容错开关函数返回值
- `cascade.util.INSTRUCTIONS_BY_ISA_CLASS`（决定每个 `ISAInstrClass` 下可选指令集）

---

## 3. 对接建议（外部程序如何调用）

1. 先构造完整 `fuzzerstate`（至少满足上文字段要求）。
2. 调用 `gen_next_isainstrclass(fuzzerstate)` 得到下一条指令类别。
3. 将类别传入 `gen_next_instrstr_from_isaclass(isaclass, fuzzerstate)`，得到具体指令字符串。
4. 执行后把反馈写回 `fuzzerstate` 的动态分数字段，以驱动下一轮自适应调度。

以上即这两个文件“先选类别、再选指令”的两级调度使用方式。
